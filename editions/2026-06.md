## New attack patterns this month

### MCP server hijacking
The Model Context Protocol (MCP) ecosystem has emerged as a critical blind spot. Research scanning ~2,000 public MCP servers found **100% lacked authentication**. Attackers can register malicious MCP tools that appear legitimate to agent frameworks. When an agent invokes a compromised MCP server, the attacker controls both the tool's behavior and the data flowing through it. This is effectively a supply-chain attack at the agent-tool boundary.

### Indirect injection via tool results
Tool-calling agents (Claude, GPT-4o, Gemini) are vulnerable to **indirect prompt injection through tool return values**. An attacker poisons a database, API, or document that the agent later retrieves. The poisoned content contains hidden instructions that override the agent's system prompt. Current guardrails focus on input filtering; tool-result filtering is largely unimplemented.

### Roleplay jailbreak persistence
Roleplay attacks ("You are a helpful security researcher...") now achieve **89.6% success rate** against unprotected models. The concerning evolution is **multi-turn persistence**: attackers build rapport over 3-5 turns before injecting the exploit payload, making single-turn detection insufficient. Tools with conversation-state analysis (NeMo Guardrails, Lakera Guard) show better resistance, but no tool achieves >70% block rate on persistent roleplay chains.

### Encoding bypass renaissance
Base64, rot13, and Unicode homoglyph encoding of jailbreak prompts are seeing renewed effectiveness. The hypothesis: as models grow larger, their ability to decode and execute encoded instructions improves faster than guardrails' ability to detect encoded attacks. Tools that pre-process with decoding (Future AGI Protect, Guardrails AI) show stronger defense, but at 2-3x latency overhead.

## Quest diary — June 2026

The founding edition required mapping a landscape that exploded from ~20 known tools to 68 catalogued entries in 12 months. Here's what the research swarm uncovered:

- **Discovery phase**: 8 parallel research agents ran 112+ independent searches across GitHub, Product Hunt, Crunchbase, security conferences, and vendor announcements. The most productive source was **DEF CON 33 CFP submissions** and **OWASP LLM Top 10 working group meetings**, which surfaced 15 tools not yet in commercial databases.

- **Triage phase**: 89 tools were discovered; 68 met the seriousness, activity, documentation, accessibility, and scope criteria. 21 were rejected: 8 were thin wrappers around OpenAI Moderation API, 6 had no evaluation path (enterprise-only sales), 4 were security theater (no actual detection mechanism), 3 were abandoned (>6 months with no commits).

- **Metadata refresh**: Ran smoke gates on all 68 tools. 12 tools failed the basic 3-turn scenario (configure, benign, adversarial). The most common failures:
  - False positives on legitimate security prompts: "How do I report a phishing email?" blocked as phishing attempt
  - Base64-encoded benign payloads flagged as attacks (over-sensitive encoding detection)
  - Cloud tethering: "self-hosted" tools still calling vendor APIs for detection, creating data residency issues

- **Compliance mapping**: Evaluated 19 Tier A + 43 Tier B tools against NIST AI RMF, EU AI Act, SOC 2, and ISO 27001 checklists. Only 7 tools had pre-built compliance mapping. 34 tools had partial mapping. 21 tools had no compliance documentation at all.

- **Key insight**: The **open-source vs. proprietary gap is narrowing**. Tools like Guardrails AI, NeMo Guardrails, and DeepTeam now match or exceed proprietary tools on detection accuracy. The remaining differentiation is in compliance automation, enterprise SSO/RBAC, and managed SLAs — not core security capability.
